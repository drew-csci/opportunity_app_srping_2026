# Django Overview

Django is a high-level Python web framework that encourages rapid development and clean, pragmatic design. It follows the Model-View-Template (MVT) architectural pattern, which is similar to Model-View-Controller (MVC) but with Django's own terminology. Django provides built-in features for handling common web development tasks, such as user authentication, database management, URL routing, form handling, and security.

## History of Django
Django was created in 2003 by Adrian Holovaty and Simon Willison while working at the Lawrence Journal-World newspaper. It was released as an open-source project in 2005. The framework was designed to help developers build web applications quickly and efficiently, drawing inspiration from earlier frameworks like Ruby on Rails. Django has since become one of the most popular web frameworks, powering websites like Instagram, Pinterest, and The Washington Post. It emphasizes the "Don't Repeat Yourself" (DRY) principle and includes a robust ORM, admin interface, and extensive documentation.

## How Files Interact
1. **Request Flow (MVT Pattern)**:
   - A user visits a URL (e.g., `/dashboard/`). Django's `urls.py` (root or app-level) matches it to a view in `views.py`.
   - The view (a function/class) processes the request: queries models for data, handles forms, and renders a template.
   - Templates use data from views and extend `base.html`. Static files (CSS) are linked for styling.
   - Models interact with the database (PostgreSQL via settings and `.env`).

2. **Database and Migrations**:
   - Models define tables. Run `manage.py makemigrations` to create migration files, then `manage.py migrate` to apply them.
   - Apps like `accounts` and `pages` have their own migrations.

3. **Authentication**:
   - `accounts` app handles users. Views in `pages` might check authentication (e.g., `if request.user.is_authenticated`).

4. **Forms and User Input**:
   - Forms in `forms.py` validate input. Views use them to process POST requests and save to models.

5. **Testing and Deployment**:
   - `test.py` or `tests.py` run tests. `requirements.txt` ensures dependencies. `bootstrap.py` might automate setup.

6. **Scripts and Outputs**:
   - `pack_dir_to_xml.py` generates `output_20251011.xml`, possibly for exporting the project structure.

To run the app: Set up a virtual environment, install requirements, run migrations, and start with `manage.py runserver`. The `.env` configures the remote DB.
