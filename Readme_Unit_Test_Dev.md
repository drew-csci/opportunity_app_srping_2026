# Unit Test README

## Purpose

This document explains the new unit test added for the organization dashboard flow in the Opportunity App.

The selected test verifies that the organization dashboard includes the `Post New Opportunity` action and that the action points to the correct route.

---

## Unit Test Added

File:
[pages/tests.py](/Users/devhirpara/Opportunity/opportunity_app_srping_2026/pages/tests.py)

Test name:
`test_organization_dashboard_contains_post_new_opportunity_link`

---

## What This Test Checks

This test confirms three important behaviors:

1. An organization user can successfully open the organization dashboard.
2. The dashboard page shows the text `Post New Opportunity`.
3. The dashboard contains the correct link to the opportunity creation route:
`/org/opportunities/new/`

This is important because the issue requirement says that an organization user should be able to go to the dashboard and see an option to post a new opportunity.

---

## How The Test Works

The test follows this flow:

1. It creates a test organization user in `setUp`.
2. It logs in as that organization user using Django’s test client.
3. It sends a `GET` request to the organization dashboard route.
4. It checks that the page returns HTTP status `200`.
5. It checks that the response contains the visible text `Post New Opportunity`.
6. It checks that the response contains the URL generated by:
`reverse('organization_post_opportunity')`

This makes sure both the user-facing label and the destination route are correct.

---

## Why This Test Is Useful

This test helps protect the dashboard feature from regressions.

If someone later changes the dashboard template or routing by mistake, this test can catch problems such as:
- the posting button being removed
- the text label changing unexpectedly
- the link pointing to the wrong page
- the organization dashboard failing to load correctly
